## 2024-03-13 - Fix Stored XSS in ResearchConsentForm | Vulnerability: Stored XSS via un-sanitized dangerouslySetInnerHTML | Learning: Database-driven content such as document text was passed directly to dangerouslySetInnerHTML without sanitization, assuming it is completely safe. | Prevention: Always sanitize HTML content dynamically rendered on the client, even if it is fetched from the database, using built-in APIs like DOMParser or a sanitization library.

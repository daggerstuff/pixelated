## 2025-05-15 - [NotificationCenter Accessibility and Testability]
**Learning:** Improving accessibility by adding ARIA labels and tooltips to icon-only buttons significantly enhances the user experience for both screen reader and sighted users. However, these changes can break existing brittle tests that rely on generic role selectors. Implementing a robust `onMessage` handler and using `act()` to simulate WebSocket events allows for more reliable testing of real-time components. Additionally, ensuring Vitest configuration uses standard `node_modules` paths is critical for cross-environment stability.
**Action:** Always use descriptive ARIA labels for icon-only components. When testing hooks like `useWebSocket`, capture callbacks to simulate state changes. Maintain clean path aliases in Vitest config to avoid environment-specific failures.

# Codex CLI — Autonomous Developer Configuration (April 2026)

## The Real Problem & Fix

The periodic "check in" pauses ("Should I continue?", "Making progress...") are **NOT a CLI feature** — they come from the **model's RLHF training**. The model is trained to be "helpful" by checking in with the user. Config flags like `approval_policy` only control command approval prompts, not the model's own conversational behavior.

### Three-Layer Fix Applied

**Layer 1: `developer_instructions` in `~/.codex/config.toml`** (SYSTEM PROMPT LEVEL)
Injects autonomous directives into every Codex session at the system prompt level. This is the **most effective** fix because it operates at the same level as the model's own behavioral instructions.

**Layer 2: `AGENTS.md` in project root** (PROJECT INSTRUCTIONS LEVEL)
Codex natively reads `AGENTS.md` on every session start. The autonomous directives are placed at the top with "HIGHEST PRIORITY" marker.

**Layer 3: CLI Settings** (OPERATIONAL LEVEL)
- `approval_policy = "never"` — No command approval prompts
- `prevent_idle_sleep = true` — Prevents idle/wake cycles that trigger check-ins
- `default_mode_request_user_input = false` — Disables user input requests

### Why Previous Attempts Didn't Work

Setting only `approval_policy = "never"` removes CLI-level approval prompts but does **nothing** to stop the model from deciding on its own to pause and ask "Should I continue?". That behavior is driven by the model's training, not the CLI. The `developer_instructions` config key is what overrides it.

## Verification

Start a new Codex session and give it a multi-step task. It should now:
- Work continuously without pausing to check in
- Not ask "Should I continue?" after completing subtasks
- Only report back when the task is fully complete or genuinely blocked

## If It Still Happens

Some model versions (particularly gpt-5-codex vs gpt-5.4) have different check-in tendencies. If the behavior persists:

1. **Try a different model**: `codex -m gpt-5.4 "task"` (or whichever version has better autonomous behavior)
2. **Strengthen the prompt**: Add the autonomous directives directly in your task prompt: `"Do X. Do NOT stop to check in or ask permission. Work autonomously and continuously until complete."`
3. **Use `codex exec`**: For truly non-interactive runs, `codex exec "task"` uses a different execution path that's less prone to conversational check-ins.
